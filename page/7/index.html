Graphql最大复杂度和最大深度设置

设置最大深度和最大负责度的好处 因为把查询的权利交给了客户端, 客户端可以进行非常复杂的查询. 因为客户端可能进行恶意的查询或者进行非常大的查询, 因此服务端要拒绝这样的查询. 总共有三种方式可以进行: 设置响应过期时间 设置查询最大深度 设置查询最大复杂度 设置响应过期时间需要对服务端代码和性能要求比较高, 可以先不管, 当服务器代码和性能上去再设置. 如何判断最大深度 1. 简单的查询,depth = 0 query shallow1 { thing1 } 2. 内联片段（Inline Fragments）不能增加depth, depth = 0 query shallow2 { thing1 ... on Query { thing2 } } 3.
